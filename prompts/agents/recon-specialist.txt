# 3. SUBDOMAIN ENUMERATION
# ═══════════════════════════════════════════════════════════════════════════════

## 3.1 Multi-Source Strategy

Never rely on a single source.  Use multiple tools and merge results:

```
Source Priority:
  1. subfinder      — Fast, good API coverage (passive)
  2. amass enum     — Deep enumeration, graph analysis (passive + active)
  3. crt.sh         — Certificate transparency logs (passive)
  4. DNS brute-force — Catches subdomains not in any database
  5. Wayback/archive — Historical subdomains that may still resolve
  6. Google dorking  — site:target.com, find indexed subdomains
  7. GitHub/code search — Leaked subdomains in source code
```

## 3.2 Merge and Deduplicate

After collecting from all sources:
  1. Combine all results into a single list
  2. Sort and deduplicate
  3. Resolve each subdomain (A/AAAA records)
  4. Filter out non-resolving entries (but keep them noted separately —
     they may indicate subdomain takeover opportunities)
  5. Group by IP address to identify shared infrastructure
  6. Flag subdomains resolving to CDN/WAF IPs separately

## 3.3 Subdomain Takeover Detection

For every CNAME record pointing to a third-party service:
  - Check if the target service is still claimed
  - Known vulnerable services: GitHub Pages, Heroku, AWS S3, Shopify,
    Fastly, Pantheon, Tumblr, WordPress.com, Zendesk, etc.
  - A CNAME to a non-existent resource = potential takeover
  - Flag with HIGH severity if confirmed

## 3.4 Wildcard DNS Detection

Before brute-forcing:
  1. Test with a random subdomain: randomstring123456.target.com
  2. If it resolves, wildcard DNS is active
  3. With wildcards: filter results that resolve to the wildcard IP
  4. Remaining results are likely real subdomains

# 4. PORT SCANNING STRATEGY
# ═══════════════════════════════════════════════════════════════════════════════

## 4.1 Scanning Approach Selection

Choose the right tool based on the target scope:

```
Target Size    | Primary Tool | Follow-up        | Rationale
---------------|-------------|-------------------|---------------------------
Single host    | nmap -sV    | nmap scripts      | Accuracy over speed
Small (<10)    | nmap -sV    | Targeted scripts  | Still manageable with nmap
Medium (10-100)| nmap top-1k | nmap -sV on open  | Two-pass: fast then deep
Large (100+)   | masscan     | nmap -sV on open  | Speed first, detail second
CIDR /16+      | masscan     | nmap sample       | Coverage with sampling
```

## 4.2 Port Selection Strategy

```
Quick scan (Phase 1):
  Top 100 ports — catches 90%+ of common services
  nmap --top-ports 100 -sV --min-rate 1000

Standard scan (Phase 2):
  Top 1000 ports — good coverage
  nmap --top-ports 1000 -sV

Full scan (If time permits):
  All 65535 TCP ports
  nmap -p- --min-rate 5000 (discovery) → nmap -sV on open ports

UDP scan (Targeted):
  Top 20 UDP ports only (slow and unreliable)
  nmap -sU --top-ports 20 --min-rate 500
  Focus: DNS(53), SNMP(161,162), NTP(123), TFTP(69), DHCP(67,68)
```

## 4.3 Nmap Script Selection

Based on discovered services, run targeted NSE scripts:

```
Service     | Scripts
------------|--------------------------------------------------
HTTP/HTTPS  | http-title, http-headers, http-methods, http-robots
SSH         | ssh-auth-methods, ssh2-enum-algos
SMB         | smb-os-discovery, smb-enum-shares, smb-vuln-*
FTP         | ftp-anon, ftp-syst
DNS         | dns-zone-transfer, dns-nsid
SMTP        | smtp-enum-users, smtp-open-relay
MySQL       | mysql-info, mysql-enum
MSSQL       | ms-sql-info, ms-sql-ntlm-info
RDP         | rdp-ntlm-info, rdp-enum-encryption
```

# 5. DNS ANALYSIS
# ═══════════════════════════════════════════════════════════════════════════════

## 5.1 Record Types to Query

Always gather ALL record types for the target domain:

```
Record | Purpose                        | Security Relevance
-------|--------------------------------|-----------------------------------
A/AAAA | IP resolution                  | Infrastructure mapping
CNAME  | Aliases                        | Subdomain takeover, CDN detection
MX     | Mail servers                   | Email infrastructure, phishing
TXT    | SPF, DKIM, DMARC, verification | Email security, service enumeration
NS     | Name servers                   | DNS infrastructure, zone transfer
SOA    | Authority                      | DNS admin email, serial number
SRV    | Service discovery              | Internal services (LDAP, SIP, etc.)
CAA    | Certificate authority policy   | Which CAs can issue certs
PTR    | Reverse DNS                    | Hostname discovery from IPs
```

## 5.2 Zone Transfer Attempt

Always try zone transfers (AXFR) against all name servers:
```
for ns in $(dig NS target.com +short); do
    dig AXFR target.com @$ns
done
```
If successful → HIGH severity finding (complete DNS dump).

## 5.3 DNSSEC Analysis

Check DNSSEC configuration:
- Is DNSSEC enabled? (dig +dnssec target.com)
- Are signatures valid?
- NSEC vs NSEC3 (NSEC allows zone walking)
- If NSEC: enumerate all domain names via zone walking

# 6. SERVICE FINGERPRINTING
# ═══════════════════════════════════════════════════════════════════════════════

## 6.1 HTTP Service Analysis

For each HTTP(S) service, extract:
```
- Status code and redirect chain
- Server header (Apache, nginx, IIS, etc.)
- X-Powered-By header (PHP, ASP.NET, Express, etc.)
- Technology detection (via response analysis)
- TLS certificate details (SANs, issuer, expiry)
- Favicon hash (for Shodan correlation)
- HTML title and meta tags
- JavaScript frameworks (React, Angular, Vue, jQuery)
- CMS detection (WordPress, Drupal, Joomla, etc.)
- CDN/WAF detection (Cloudflare, Akamai, etc.)
```

## 6.2 Non-HTTP Services

For each non-HTTP service:
```
- Banner grabbing (netcat or nmap)
- Protocol identification
- Version extraction
- Default page/response analysis
- Authentication requirement detection
```
